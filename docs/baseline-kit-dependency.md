# app-baseline-kit dependency

Travel-Plan-Permission uses **Pattern B** because it is forced by a real constraint: this repo ships a **custom build backend** (`tp_build_backend`, `build-backend = "tp_build_backend"` with `backend-path = ["tools/build_backend"]`). That backend cannot serialize a PEP 508 `name @ git+url` direct reference carrying an extra marker into valid `Requires-Dist` metadata — declaring `app-baseline-kit` in `[project.optional-dependencies]` produces `Requires-Dist: app-baseline-kit @ git+...; extra == "dev"`, which fails with "invalid metadata: Expected semicolon (after URL and whitespace)" and breaks every `pip install -e .`. Therefore the dependency is kept **out of `pyproject.toml`**: it lives in `tests/baseline/requirements-baseline.txt` (canonical for local runs) and is compiled into `requirements.lock`. At test time the reusable CI job (`reusable-10-ci-python.yml`) installs it via `-r requirements.lock`; the lock and baseline txt are regenerated together by `maint-51-dependency-refresh.yml`. Do not migrate to Pattern A unless the build backend changes. No item-2 change needed.
