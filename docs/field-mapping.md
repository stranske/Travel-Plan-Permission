# Organizational workbook field mapping

TPP keeps the organization-provided workbooks under the repository-level
`templates/` directory as immutable source artifacts. Runtime exports use the
packaged, normalized itinerary copy declared by the active mapping in
`src/travel_plan_permission/config/excel_mappings.yaml`.

## Version tracking

| Template file | Template ID | Status | Notes |
| --- | --- | --- | --- |
| `Travel_Itinerary_Form_Jan_1_2026_revised.xlsx` | `ITIN-2026.1` | Active | Organization source; worksheet `Itinerary Form` |
| `Travel_Itinerary_Form_Jan_1_2026_runtime.xlsx` | `ITIN-2026.1` | Runtime derivative | LibreOffice-normalized copy used to avoid the source workbook's excessive in-memory expansion |
| `travel_request_template.xlsx` | `ITIN-2025.1` | Legacy | Generated placeholder retained only for compatibility |
| `Expense_Report_Form_FY2026_revised_Jan_2026.xlsx` | Not yet mapped | Source only | Post-trip reimbursement form; not used for pre-trip itinerary approval |

The mapping metadata records SHA-256 checksums for both the organization source
and the runtime derivative. Any source workbook change requires a new template
ID, a new verified runtime copy, and updated mapping tests.

## Active itinerary mapping

The active mapping writes only organization input cells on `Itinerary Form` and
leaves gray calculated cells and workbook formulas intact.

- Traveler and trip: `traveler_name -> C6`, `destination_zip -> G6`,
  `event_dates -> M6`, `business_purpose -> C7`, and travel dates to `M7:M8`.
- Flight: airports to `D14` and `J14`; outbound details to `F16`, `I16`, and
  `K16`; return details to `F17`, `I17`, and `K17`; selected fare to `M17`; and
  the lowest comparison fare to `E20`.
- Airport parking: the total estimate is converted to a daily rate in `K22`;
  the organization formula in `K23` calculates the total from trip days.
- Hotel: name, rate, address, and city/state use `C36`, `K36`, `C38`, and `H38`.
  Comparison notes use `K44`; two comparable hotels use rows 45 and 46.
- Meals: eligible breakfast, lunch, and dinner counts use `H52:H54`; per-diem
  rates and totals remain formula-driven.
- Ground transport: the selected option is encoded in `M73:M77`, and its
  estimate is written only to the corresponding cost cell.
- Affirmations: fare-threshold, hotel-comparison, hotel-selection, and budget
  confirmations use the form's checkbox cells and `☒` / `☐` values.

The organization form has no cost-center field. That intentional omission is
declared in mapping metadata rather than silently dropping the canonical value.
Destination city/state, hotel nights, parking totals, and ground-transport
summary values are derived by the organization workbook and are likewise
declared as derived fields.

## Runtime behavior

- Dates and flight timestamps are written as typed Excel dates/times, not text.
- Currency inputs are rounded to two decimals and retain currency formatting.
- The worksheet is selected by its declared name; generation fails clearly if
  the organization renames or removes it.
- Workbook calculation is set to automatic with a full recalculation on open.
- Generation patches only the named worksheet's declared input cells in a new
  OOXML ZIP stream. All other workbook members are copied byte-for-byte, so the
  organization layout is preserved and neither source nor runtime template is
  modified in place.
- The Washington, DC business fixture exercises the complete applicable path
  through `tests/python/test_spreadsheet_fill.py` and stops before submission.

## Output bundle

- `itinerary.xlsx`: populated organizational workbook tagged with the active
  template version.
- `summary.pdf`: quick-review summary generated by the existing portal flow.
- `conversation_log_json`: serialized intake answers and attachment references.
- Optional attachments, such as a conference or meeting agenda, remain separate
  artifacts referenced by the approval packet.
